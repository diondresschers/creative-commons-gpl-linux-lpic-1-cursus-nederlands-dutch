static library

  • Een stukje code dat gebruikt wordt door een programma
  • De static library is deel van dat programma
  • De code wordt niet gedeeld met andere programma's
LPIC-1 | Section 102.3 | Page 3/16

shared library

  • Een shared library is een stukje code dat door verschillende programma's gebruikt kan worden
  • Van in het geheugen geladen shared libraries kunnen meerdere programma's (tegelijkertijd) gebruik maken.
  • Vergelijkbaar met een Windows dll-file.
LPIC-1 | Section 102.3 | Page 4/16

ldd

  • Met 'ldd' laat je van een programma de gedeelde (shared) libraries zien

LPIC-1 | Section 102.3 | Page 6/16

/etc/ld.so.cache

  • Dit is de cache van alle shared libraries, die programma's kunnen gebruiken
  • Programma's weten hun shared libraries hierdoor snel(ler) te vinden
LPIC-1 | Section 102.3 | Page 7/16

/etc/ld.so.conf

  • De /etc/ld.so.conf file bepaalt welke programma's in de cache komen
  • Deze kan verwijzen naar de /etc/ld.so.conf.d directory
LPIC-1 | Section 102.3 | Page 8/16

/etc/ld.so.conf.d

  • In de /etc/ld.so.conf.d staan gegroepeerde files
  • Deze files kunnen op hun beurt weer verwijzeren naar shared libraries (so files)
  • Maar ze kunnen ook weer verwijzen naar directories met daarin de so-files.

ldconfig

  • 'ldconfig' herlaadt de locaties van de shared libraries in de /etc/ld.so.cache
  • Dit commando wordt door programma's uitgevoerd nadat je het geintalleerd hebt.
  • Je kan 'ldconfig' ook zelf starten.

LD_LIBRARY_PATH

  • Voeg via de enviroment variabele LD_LIBRARY_PATH locaties toe aan je shared libraries
  • Deze worden eerst geladen en hebben prioriteit boven latere shared libraries
  • Handig om code/libraries te testen, echter niet adviseerbaar.

Vraag 2

Which of these commands allows you to use shared libraries that are in /usr/local/lib?


  • export LD_PRELOAD=/usr/local/lib
  • export LD_LIBRARY_PATH=/usr/local/lib
  • ldconfig /usr/local/lib
  • ldd /usr/local/lib
