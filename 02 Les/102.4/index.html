dpkg

  • dpkp is de debian package management tool
  • Via 'dpkg' wordt er geen rekening gehouden met software afhankelijkheden (dependencies).
  • Dependencies zijn gedeelde stukken software die een of meerdere andere pakketten nodig hebben om te kunnen werken (anders dan shared libraries dat gebruikt maakt van gedeeld geheugen).
LPIC-1 | Section 102.4 | Page 3/29

dpkg

  • Je wordt op de hoogte gebracht van eventuele 'dependencies', maar deze niet voor je geinstalleerd.
  • Als een dependency niet meer klopt (dependency hell) dan draait je software niet meer goed.
  • De optie '-l' geeft je een lijst met geinstalleerde software.
  • Via '-p' kan je meer informatie printen over dat software pakket.
  • 'dpkg' kan '*.deb' (debian) files installeren.

apt-get

  • 'apt-get' is een onderdeel van de advanced package tool
  • Er wordt wel rekening gehouden met dependencies (in tegenstelling tot dpkg)
  • De dependencies worden ook voor je mee geinstalleerd
  • Om het besturingssysteem te upgraden (dus packages bij te werken naar een nieuwere versie) gebruik je: apt-get upgrade
  • Good common practice is om voor de 'upgrade' een 'update' te doen.
LPIC-1 | Section 102.4 | Page 10/29

apt-get

  • Om een programma te installeren gebruik je 'install'. Gebruik de optie '-y' om op elke vraag bij de installatie 'Yes' te antwoorden.
  • Om een pakket te verwijderen gebruik je 'remove'
  • 'apt-get' haalt packages van het internet, uit zogenaamde repositories. Je kan de lijst updaten door 'update'

apt-cache

  • Met apt-cache wordt in de lokale files gezocht.
  • Je kan zo met 'search' zoeken of er programma's zijn die je wilt installeren.
  • Je kan de dependencies van het pakket zien via 'depends'.
  • Met 'rdepends' kan je zien welke pakketten allemaal gebruik maken van een bepaalde dependency.

aptitude

  • Als je niet weet of je 'apt-get' of 'apt-cache' moet gebruiken, gebruik dan 'aptitude'.
  • Dit pakket wordt echter niet meer standaard op Ubuntu mee geinstalleerd.
  • Alleen 'aptitude' dus zonder opties geeft een mooie TUI (Terminal User Interface) waar je zelf je pakketten kan selecteren via een menu.

/etc/apt/sources.list

  • Alle bronnen waarvan 'apt-get' de packages vandaan haalt staan in de zogenaamde repositories.
  • Een repository is een collectie software die bijgehouden wordt door of de distributie leverancier of individuele software ontwikkelaars.
  • De repositories zijn te vinden in de file '/etc/apt/sources.list'.
  • Het voordeel van de repositories is dat je programma's via 'apt-get' kan installeren en niet afhankelijk bent van 'dpkg'

Vraag 1

You have not run apt-get on a system for some time, but it has been run on the system before. What apt-get command would you run to download the latest list of packages, but not the packages themselfes?


  • apt-get build-dep
  • apt-get mirror-select
  • apt-get update
  • apt-get upgrade
  • apt-get dist-upgrade
