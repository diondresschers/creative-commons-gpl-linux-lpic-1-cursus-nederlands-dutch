df

  • df (disk free) laat je zien hoeveel ruimte er nog is op je gemounte opslag ruimtes
  • -h (human readible) zorgt dat het beter leesbaar is (in Mega en Giga bytes).
  • Je kan ook mount aangeven na het commando als je alleen deze wilt zien (dus bv 'df boot').
LPIC-1 | Section 104.2 | Page 3/38

df

  • Optie '-i' laat het aantal inodes zien dat er in gebruik is.
  • Elke file heeft een unieke inode
  • Via die inode weet Linux die file op de harde schijf terug te vinden.
  • Elke inode kost ook wat ruimte en je kan op een gegeven moment door al je inodes heen zijn.
  • Je kan het aantal inodes meestal verhogen.

du

  • du (disk usage) geeft disk usage.
  • Het commando laat standaard het aantal bytes zien per directory.
  • Alleen het commando geeft resultaten van de huidige directory.
  • '-h' maakt het weer 'human readable'
  • '-d' (depth) laat zien hoe diep er gezocht kan worden.

fsck

  • fsck (file system check) controleert het filesysteem
  • Met de optie '-t' kan je het type meegeven (bv. ext3)
  • Je kan ook bv. het commando 'fsck.ext3' draaien.
  • Je krijgt een melding als het filesysteem nog gemount is (en het wordt dan afgeraden om deze tool te draaien).
  • Draai deze tool ook als 'root'.

debugfs

  • 'debugfs' is een tool om het filesysteem te debuggen.
  • Het geeft een interactief menu.
  • Gebruik de optie '-w' write om niet alleen te debuggen maar ook om te wijzigen.
  • Uiteraard 'sudo' rechten.
  • 'help' in het menu geeft de hulp informatie.

dumpe2fs

  • 'dumpe2fs' dumpt info over het EXT(2) filesysteem
  • Het geeft onder andere info over de mount, het UUD, de block size en het aantal inodes.
  • Vergeet 'sudo' weer niet
  • Het filesysteem hoeft niet geunmount te worden.

mke2fs

  • 'mke2fs' is in veel gevallen hetzelfde programma als 'mkfs'
  • 'mke2fs' (make EXT(2) filesystem) maakt een filesysteem op een partitie.
  • Als er een verschil is tussen 'mkfs' en 'mke2fs' (als het geen symbolic link is) dat 'mke2fs' alleen geschikt is voor 'EXT' filesystemen.
