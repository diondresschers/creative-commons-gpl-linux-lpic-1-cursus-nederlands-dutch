LPIC-1 | Section 104.6 | Page 2/16

inodes

  • Om te begrijpen wat hard en symbolic links zijn, is het goed om te weten wat inodes zijn.
  • Via inodes (index node) worden op een filesysteem bijgehouden waar welk object (bestand) waar staat.
  • De inode is eigenlijk de metadata van een file (behalve de bestandsnaam).
  • De inode bevat ook een link waar het de data kan vinden op het filesysteem (verdeeld over verschillende blocks).
  • De inode wordt weergegeven door een decimaal nummer (het zogenaamde inode nummer).
  • Via 'df -i' krijg je informatie over het aantal inodes op je fileysystemen.
LPIC-1 | Section 104.6 | Page 3/16

inodes

  • Je kan via 'ls -i' het inode nummer vinden van files.
  • Aangezien een bestandsnaam niet bepaald is in de inode kan je meerdere bestandsnamen linken naar hetzelfde bestand.
  • Met 'stat' en dan de filename kan je meer in de metadata kijken van die inode (dus van de file zelf). Veel van deze metadata is ook te zien via 'ls -li'.
  • Ook directories bevatten een inode (nummer).
  • Je ziet dat de hidden files '.' en '..' een inode nummer hebben en het blijken de huidige en de bovenliggende directories te zijn.
LPIC-1 | Section 104.6 | Page 4/16

inodes

  • Informatie die een inode bevat:
  • Rechten/Permissions
  • Groote van de file
  • User ID
  • Group ID
  • Device waar de file staat
  • Locatie van de file op dat device (een link, niet alle blokken worden benoemd)
  • Access-tijd
  • Modify-tijd
  • Change-tijd (dit is de tijd dat de metadata aangepast wordt)

ln

  • Een inode is niks meer dan de metadata naar een file op de harde schijf, echter zonder de naam.
  • We kunnen met het 'ln' (zonder opties) een zogenaamde 'hardlink' aanmaken dit is nieuwe bestandsnaam die verwijst naar een bestaande inode.
  • Deze inode verwijst op zijn beurt weer naar de data op de harde schijf.
  • Het inode nummer van het originele bestand is dan ook hetzelfde als het inode nummer van de 'hardlink'.
  • Veranderen we de inode metadata of de inhoud van dat bestand, dan geldt dat dus voor beide (hard)links.

ln

  • Wederom een inode is niks meer dan de metadata naar een file op de harde schijf, echter zonder de naam.
  • We kunnen ook een softlink aanmaken met 'ln -s'. Nu wordt niet verwezen naar hetzelfde inode nummer als bij de hardlink, maar we verwijzen nu door naar de naam die verwijst naar de inode.
  • De softlink krijgt in tegenstelling naar de hardlink wel een nieuw inode nummer.
  • Je kan een softlink min of meer vergelijken met een snelkoppeling (of shortcut) in 'Windows'.
