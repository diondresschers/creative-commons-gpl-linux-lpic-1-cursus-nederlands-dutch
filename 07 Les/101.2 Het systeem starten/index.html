bios

  • In de BIOS Basic In Out System kan je wijzigingen doen van je hardware (dit is geen onderdeel van het Operating System).
  • De BIOS wordt als eerste gestart als je een PC aanzet.
  • Het belangrijkste doel is om de aanwezige hardware in het systeem te testen en te initialiseren.
  • Op het laatst verwijst de BIOS weer door naar het MBR (Master Boot Record) op de disk die dan verder de software op de schijf laadt.
  • De BIOS wordt langzaam vervangen door (U)EFI

bootloader

  • De bootloader staat op de harddisk en wordt aangeroepen via het MBR (staat in het begin van een partitie).
  • Het doel van een bootloader is om het besturingssysteem op te starten
  • De bootloader wordt opgestart nog voordat de (Linux) kernel gestart is, dus het is geen onderdeel van Linux (wel van een GNU/Linux distributie).
  • Bootloaders (of beter gezegd Bootmanagers) die we onder andere kennen zijn LILO (oud) en GRUB (nieuw)
LPIC-1 | Section 101.2 | Page 5/18

bootloader

  • Druk je de <ESC>-toets kan je in het GRUB bootloader menu komen (mocht deze worden overgeslagen).
  • Bij een bootloader zoals GRUB kan je kiezen welk besturingsysteem of met welke kernel gestart wordt.
  • Met de <E>-toets kan je regels tijdelijk editen, om het opstarten te beinvloeden.
  • Zo kan je bij edit kiezen voor 'quiet' (weinig info) of 'splash' (mooi grafisch scherm).

kernel

  • De kernel is de basale kern van een besturingssysteem.
  • De kernel geeft de gebruiker via applicaties toegang tot de processor, geheugen en apparaten maar is zelf het minst zichtbaar aanwezig.
  • De applicaties vallen niet onder de kernel, dus ook niet CLI applicaties zoals 'cp', 'pwd' en 'cat'
  • Linux is geen besturingssysteem maar is de kernel van een besturingssysteem.
  • De applicaties bestaan/bestonden grotendeels uit applicaties van het GNU besturingssysteem.

dmesg

  • Het bestand '/var/log/dmesg' laat een lijst met gebeurtenissen zien van het opstarten.
  • Het commando 'dmesg' laat echter ook de meldingen zien nadat er is opgestart.
  • De optie '-k' laat je alleen kernel messages zien.
  • De optie '-D' (Disable) zorgt ervoor dat meldingen niet meer in de console zichtbaar zijn.

init

  • Init is het eerste proces start gestart wordt tijdens het starten van het OS.
  • Init blijft draaien totdat het systeem weer afgesloten wordt.
  • Init heeft atlijd een Process ID van 1
  • Alle andere processen vallen weer onder 'init'.
  • Naast dat 'init' een proces is, is het ook weer een commando, waarmee je oa. runlevels kan veranderen.
  • Init wordt langzaam vervangen door het nieuwere 'upstart'

Vraag 2

The message "Hard Disk Error" is displayed on the screen during Stage 1 of the GRUB boot process. What does this indicate?


  • The kernel was unable to execute /bin/init
  • The next Stage cannot be read from the hard disk because GRUB was unable to determine the size and geometry of the disk
  • One or more of the filesystems on the hard disk has errors and a filesystem check should be run
  • The BIOS was unable to read the necessary data from the Master Boot Record to begin the boot process.
