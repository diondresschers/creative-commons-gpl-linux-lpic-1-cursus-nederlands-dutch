runlevel

  • Een linux distro kan draaien in een bepaalde 'runlevel'
  • Deze lopen van 0 t/m 6
  • Aan elke runlevel hangen bepaalde eigenschappen zoals: wel of geen netwerk, wel of geen GUI, shutdown of reboot.
  • Elke distributie heeft zijn eigen runlevel indeling.
  • In '/etc/inittab' kan je vaak je standaard runlevel waarde zien (aleen bij gebruik 'init' niet bij 'upstart').
LPIC-1 | Section 101.3 | Page 3/22

runlevel

  • Hier de indeling volgens de LSB (Linux Standard Base:
  • 0 Halt - Schakel het systeem uit
  • 1 Single user mode (CLI, geen netwerk, vaak root)
  • 2 Muti user mode (CLI, geen netwerk)
  • 3 Normaal (Multu user, CLI, met netwerk)
  • 4 Niet gebruikt (of zelf definieerbaar)
  • 5 Normaal met Grafische Interface
  • 6 Reboot - Herstart

init

  • Init is niet alleen het eerste proces dat de computer start en de vader van alle processen, maar het kan ook als commando gebruikt worden.
  • Het 'init' commando (net als 'telinit') verandert het runlevel.
  • Geef achter het 'init' commando het nummer van de rootlevel dat je wilt kiezen.
  • In de directory '/etc/init' staan allemaal scripts die 'init' gebruikt.
  • In '/etc/rc.d' zitten directories met de runlevels, in iedere directory worden scripts met een prioreit gestart of gestopped (bv 'S10network' van Start of 'K92iptables' van Kill).

/etc/init.d

  • In de '/etc/init.d' staan scripts of services zoals 'network', 'reboot' en 'iptables'
  • Deze scripts kunnen gestart zijn bij het opstarten, maar kan je ook manueel starten, herstarten of stoppen.
  • De opties die je hebt kan je zien door gewoon het script uit te voeren zonder opties.
  • In plaats van het hele pad naar het script in te voeren bv. '/etc/init.d/networking' kan je ook volstaan met het 'service' commando, dus 'service networking'
  • Als je bv. de Apache Web Server installeert komt deze ook in deze direcory erbij te staan.

telinit

  • Niet alleen met het commando 'init' kan je het runlevel wijzigen, maar ook met 'telinit'
  • 'telinit' is dus niet het 'init' proces maar een tooltje om het runlevel te wijzigen.
  • In de binary file /var/log/wtmp wordt de wijziging van de runlevels toegevoegd.
  • Met 'runlevel' kan je het huidige runlevel zien.

shutdown

  • Met 'shutdown' kan je het systeem down brengen.
  • 'shutdown' verwacht echter wel een tijd (bv 'now' of '+10' minuten)
  • Met de optie '-c' van cancel kan je een shutdown alsnog annuleren
  • De optie '-r', zorgt voor een reboot.
  • De optie '-h', zorgt voor een hhalt.
  • 'shutdown -r now' is hetzelfde als het commando 'reboot'.

Vraag 1

You are having some trouble with a disk partition and you need to do maintenance on this partition but your users home directories are on it and several are logged in. Which command would disconnect the users and allow you to safely execute maintenance tasks?


  • telinit 1
  • shutdown -r now
  • killall -9 initrd
  • /bin/netstop --maint
  • /etc/rc.d/init.d/network stop
