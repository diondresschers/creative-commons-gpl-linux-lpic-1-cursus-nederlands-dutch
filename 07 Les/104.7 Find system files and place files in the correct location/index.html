FHS

  • FHS (File Hierarchy Standard) definieert een standaard die wordt gebruikt voor de directory-structuren van unix systemen.
  • De folders die je in linux-systemen vindt zijn allemaal volgens de File Hierarchy Standard en hebben allemaal een betekenis en doel.
  • Wil je meer informatie of wil je weten hoe zo'n structuur uitziet zie FHS (zie kopje 'Bestandssysteem')
LPIC-1 | Section 104.7 | Page 3/26

find

  • Met de opdracht 'find' kan je in realtime bestanden opzoeken die voldoen aan specifieke kenmerken.
  • De expressie die bij 'find' wordt gegeven kan bestaan uit: opties, tests en acties.
  • Bij het opzoeken moet vooraf aangegeven worden waar er gezocht moet worden zoals: 'find / [expressie]'
  • Het kan erg lang duren met het zoeken met find, geef specifieke directory mee om dit te verkorten.
  • De opdracht 'find' is erg krachtig, je kan zo specifiek zoeken als je wil, zoals op tijd, permissies, users enz. Voor meer info. zie find

locate

  • Met 'locate' kan je zoeken in een database naar bestanden met een bepaalde naam.
  • 'locate' zoekt dus niet in de harde schijf, dit in tegenstelling tot wat 'find' wel doet.
  • Door gebruik te maken van een database krijg je met 'locate' sneller resultaten dan met 'find'.
  • Ook als je slechts een gedeelte van een bestand zoekt vind 'locate' dit.
  • 'locate' kan ook verborgen bestanden vinden.
  • Als je nieuwe bestanden niet kan vinden, dan kan het zijn dat de database niet up to date is. Deze is te updaten met de opdracht 'updatedb'

updatedb

  • Met de opdracht 'updatedb' kan je de (zoek)database updaten waar 'locate' gebruik van maakt, je kan hiermee ook een database aanmaken.
  • Voordat je 'locate' gebruikt is het verstandig om eerst de database te updaten.
  • Je kan 'updatedb' alleen gebruiken als root user.
  • Met de optie -o kan je de update wegschrijven naar een andere database file i.p.v. de default database.
  • De default database is '/var/lib/mlocate/mlocate.db' (Ubuntu)

/etc/updatedb.conf

  • In /etc/updatedb.conf kan zien in welke folders en filesystemen 'locate' allemaal zoekt.
  • Je kan in updatedb.conf configureren waar 'locate' mag of kan zoeken, met PRUNEFS kan je filesystemen uitsluiten en PRUNEPATHS kan je mappen uitsluiten.

whereis

  • Met de opdracht 'whereis' kan je de locatie achterhalen van bepaalde bestanden.
  • 'whereis' kan ook zoeken naar programmabestanden, sourcecodes en de man-pages van een opdracht.
  • Optie '-b' laat alleen programmabestanden zien, '-m' zoekt alleen naar man-pages en '-s' zoekt bronbestanden op.

type

  • Opdracht 'type' laat zien wat voor type (soort) programma een programma is.
  • Als je bijvoorbeeld 'type firefox' intypt dan krijg je als resultaat 'firefox is /usr/bin/firefox'
  • En als je 'type cd' intypt dan krijg je als resultaat 'cd is a shell builtin'
  • Zo kan je zien dat de opdracht 'type' verschillende informatie kan geven over programma's.
  • En bij 'type ls' krijg je als resultaat 'ls is aliased to `ls --color=auto', dus hier geeft 'type' aan dat 'ls' een alias is. Uiteindelijk wordt het programma '/bin/ls' uitgevoerd met de optie '--color=auto'.
