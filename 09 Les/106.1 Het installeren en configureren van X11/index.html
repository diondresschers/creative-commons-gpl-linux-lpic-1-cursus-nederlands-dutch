X

  • Linux en Unix zijn begonnen als Command Line Interface besturingssystemen.
  • Op een gegeven moment is daar een grafische schil bovenop gekomen.
  • De toepassing 'X Window System' (oftewel X11 of X) zorgt voor de basis van een grafische bitmap weergave.
  • X zorgt dat vensters worden weergegeven en gesleept kunnen worden.
  • Daarnaast verzorgt X de interactie met die gebruiker via een muis, toetsenbord en andere invoer apparaten.
  • Met CTRL+ALT+F1 t/m F7 kom je in TTY 1 t/m 7 die CLI of Grafisch kunnen zijn.

/etc/x11/xorg.conf

  • Je kan X configureren (met 'X -configure'), dan moet X echter niet draaien.
  • Dit kan door een runlevel te kiezen zonder GUI
  • De configuratie staat standaard in /etc/X11/xorg.conf
  • Nieuwe systemen kunnen andere locaties gebruiken (bv. /usr/lib/X11/xorg.conf.d/ voor Ubuntu)
  • Bij het uitvoeren van 'X -configure' worden er standaard config files aangemaakt die je zelf kan modificeren.
LPIC-1 | Section 106.1 | Page 5/24

/etc/x11/xorg.conf

  • Op oudere systemen zou je deze xorg.conf file kunnen tegenkomen. Zie ook x.org
  • De file is onderverdeeld in secties zoals: Device, Monitor, InputDevice, Screen, Vendor
  • Je kan zaken aanpassen zoals resoluties, muisinput tweaks en locaties van XFonts
  • Vroeger was de XFont Server een echte server, nu wordt in de xorg.conf file lokaal verwezen in de sectie 'Files'
  • Onder 'Device' vind je meer info over de videokaart zoals de PCI bus, de driver
  • Bij de Modules vind je elementen die geladen zijn zoals 'glx' (de Open Graphics Library is voor bv. games).

DISPLAY

  • Standaard draait er op een desktop 1 display manager
  • Het nummer kan je vinden in de variabele 'DISPLAY'
  • Dit zit gewoon in je 'env'
  • Je kan met 'startx' een tweede display manager starten.
  • Hiermee kunnen meerdere personen grafisch inloggen op dezelfde pc
  • In die sessie zie je dus via 'echo $DISPLAY' de initiele waarde +1

DISPLAY

  • We gaan naar TTY6 met CLI (EIGEN RISICO!!!)
  • Via CTRL+ALT+F6 gaan we dus naar TTY6
  • We starten een tweede xserver op via 'startx' (of beter 'startx -- :1, want hij kan de huidige :0 niet dubbel starten)
  • Hij start deze niet op in TTY6, maar in TTY8 (de eerstvolgende die 'vrij' is)
  • Via CTRL+ALT+F8 gaan we dus naar TTY8
  • echo $DISPLAY # Deze zal weergeven dat het niet DISPLAY :0 is, maar DISPLAY :1
  • gedit # zal de een grafisch GNOME venster starten in DISPLAY :1 en zichtbaar zijn.
  • We veranderen de variabele 'DISPLAY=:0' en daarna 'export DISPLAY'. 'gedit' start nu dus niet in DISPLAY :1, maar in DISPLAY :0 (en dus in TTY7)

xhost

  • Het commando 'xhost' laat zien wie toegang heeft tot de xserver
  • Niet alleen de lokale computer kan gebruik maken van de xserver, maar ook remote systemen.
  • Via 'xhost' kan je dan zien welke IP addressen of users toegang hebben.
  • Met 'xhost +10.0.1.5' voeg je IP adressen toe.
  • Weghalen doe je met een minteken, dus 'xhost -10.0.1.5'
  • Gebruikers toevoegen gaat bv. met 'xhost +SI:localuser:frits'

Vraag 2

Why is the xhost program considered dangerous to use?


  • It makes it difficult to uniquely identify a computer on the network.
  • It allows easy access to your X server by other users
  • It logs sensitive information to syslog.
  • It makes your computer share network resources without any authentication.
  • It is a graphical DNS tool with known exploits.
