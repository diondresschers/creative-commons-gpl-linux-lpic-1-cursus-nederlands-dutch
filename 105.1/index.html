/etc/profile

  • De globale file '/etc/profile' zet variabelen en laadt verschillende andere files, zodat daarmee je bash omgeving, met functies, aliassen en prompt ingesteld wordt.
  • Zo wordt door deze file de hele /etc/profile.d' directory afgegaan, wordt gekeken naar de globale file '/etc/bashrc' en wordt gekeken in de homedirectory van de gebruiker voor andere scripts.
  • De '/etc/profile' is geschreven in bash en door het volgen van deze config zul je de verwijzingen zien naar de andere files.
  • Welke files weer aangeroepen hangt af van het bash script en deze is weer distributie afhankelijk.

~/.bash_login

  • De laatste file '~/.bash_login' staat waarschijnlijk niet standaard in je homedirectory.
  • Je kan deze file zelf aanmaken en wordt uitgevoerd op het moment dat je inlogt.

alias

  • Je kan een alias aanmaken voor commando's en opties de je vaak gebruikt.
  • Als je de naam van je alias kort maakt, kan je dat veel typwerk en denkwerk schelen.
  • Je kan alle aliassen zien door 'alias' in te voeren.
  • Zo blijkt het commando 'll'' (voor Long Listing) geen commando te zijn maar een alias voor 'ls -l'.
  • Wil je je alias bewaren voor na een reboot, sla deze dan op in ~/.bashrc

function

  • Een function is een klein bash programmaatje dat je kan schrijven om taken makkelijker te maken.
  • Je kan zo'n complexe programma's maken als je wilt (in tegenstelling tot alias wat zeer bepertk is).
  • Zowel als de alias, plaats je deze in je ~/.bashrc file.
  • In de .bashrc file moet je je wel houden aan de functie format (je kan niet zomaar bash beginnen te typen).

lists

  • In Linux commando's kan je de volgende leestekens tussen twee (of meer) commando's terugvinden: '&&', '||', ';' en '&'
  • De '&&' is een AND teken, dit betekent als het eerste commando waar is, dan wordt ook het tweede commando uitgevoerd.
  • Het '||' teken is een exculieve OF teken, het tweede commando wordt alleen uitgevoerd als het resultaat van het eerste onwaar is.
  • Je kan controlleren of het resultaat van een command waar of niet waar is door na het commando 'echo $?' te geven.
  • Is het resultaat '0' dan is het commando waar, in alle andere gevallen is het onwaar.
LPIC-1 | Section 105.1 | Page 27/47

lists

  • Het ';' teken is om na het eerste commando, direct een tweede te kunnen ingeven. Je hoeft dus geen enter te geven en het tweede commando erna te typen.
  • Een '&' kennen we al van het in de achtergrond draaien van een commando, achter dit teken kan je meteen een tweede commando geven.

env

  • 'env' laat alle environment variabelen zien.
  • Variabelen zijn gedeeltes in het geheugen waar je data in kan zetten.
  • Je kan de variabele zelf een naam geven en de inhoud zetten.
  • Een normale variabele werkt alleen maar in de bash shell zelf en is niet bekend andere opgestarte shells.
  • De environment variabele zijn wel bekend en bereikbaar voor andere shells (mits het dezelfde gebruiker is).
