at

  • Met 'at' kan je taken voor een gebruiker schedulen
  • Je geeft de tijd (en of datum) aan na het commando.
  • Daarna geef je je reeds gemaakte script aan, of je maakt een script ter plaatse.
  • Met de CTRL+D toetscombi kan eindig je het 'at'-commando en kan je verder werken achter de commandoregel.
  • Het script wordt automatisch uitgevoerd en je kan de uitput zien in je mail.

/etc/at.deny

  • Normaal mag je als gebruikers 'at' jobs aanmaken
  • De file ''/etc/at.deny' is normaliter leeg in een linux systeem.
  • Je kan in deze file wel gebruikers toevoegen die geen 'at' mogen draaien.

/etc/at.allow

  • Gebruikers in /etc/at.allow mogen wel 'at' gebruiken
  • Deze file is standaard niet aanwezig.
  • Maak je deze file zelf aan en voeg je hier gebruikers toen dan mogen alleen deze gebruikers 'at' gebruiken
  • Bestaat deze file, dan mogen gebruikers die niet in deze file staan 'at' automatisch niet gebruiker! (behalve root).
  • Sta je zowel in de '/etc/at.allow' en '/etc/at.deny' dan heeft '/etc/at.allow' voorrang

LPIC-1 | Section 107.2 | Page 19/42

/etc/crontab

  • #MI HO DA MO DW USER COMMAND
  • '05 * * * *' is elke 5 over (van elk uur, elke dag, elke maand, elke dag van de week)
  • '00 15 * * *' is elke dag om 15:00
  • '15 05 * * 0' is elke zondagnacht om 05:15
  • '55 23 5 6 * 5' is elke juni om 23:55
  • 05-10 (5, 6, 7, 8, 9, 10)
  • */2 (is letterlijk 1/2)
  • 10,30,55 (10, 30 en 55)
  • sun-tue (van zondag t/m dinsdag)
  • jan-jul (van januari t/m juli)

crontab

  • 'crontab' is niet alleen een file als in '/etc/crontab', maar is ook een commando
  • Het bestand '/etc/crontab' is alleen te benaderen door root. Alle andere gebruikers kunnen hun eigen persoonlijke crontab-file aanmaken via het commando 'crontab'.
  • Gebruik de optie '-e' om je persoonlijke crontab te editeren.
  • Het verschil met de '/etc/crontab' is dat je bij 'crontab' geen gebruiker opgeeft (want het is al onder je eigen naam.

/etc/cron.deny

  • Zoals er de '/etc/at.deny' file is, is er ook een '/etc/cron.deny'
  • Gebruikers die hierin staan mogen geen 'crontab' commando uitvoeren
  • Standaard bestaat deze file in Linux, maar is deze leeg.
  • Wil je gebruikers weigeren zet dan de desbetreffende gebruikersnaam er in (een per regel)

/etc/cron.allow

  • Standaard bestaat de file /etc/cron.allow niet op Linux machines.

/etc/cron.*

  • Naast '/etc/crontab' en 'crontab' heb je ook nog verschillende cron folders waar je direct scripts in kan plaatsen.
  • Voorbeelden van deze folders zijn: '/etc/cron.hourly', '/etc/cron.daily', '/etc/cron.weekly' en '/etc/cron.monthly'
  • Je hoeft hier dus geen gebruiker in te zetten of een tijdsaanduiding, het script loopt automatisch zoals aangegeven in de folderbenaming.

LPIC-1 | Section 107.2 | Page 37/42

anacron

  • Anacron is bedoelt voor workstations zoals laptops en desktops.
  • Mocht een cron job niet uitgevoerd worden omdat de computer uit staat, dan zorgt anacron er voor dat deze alsnog uitgevoerd wordt op het moment dat de computer weer aan staat.
  • Er is geen aparte daemon voor nodig, anacron maakt gebruik van de huidige cron configuraties.
  • Sommige configuratiefiles/scripts beginnen met een '0' zodat deze als eerste gedraait worden (alfabetisch) voor andere scripts.
