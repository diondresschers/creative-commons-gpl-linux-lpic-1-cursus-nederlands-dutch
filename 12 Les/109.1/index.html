netwerken

  • Het netwerk dat we gebruiken is van de TCP/IP stack.
  • Hierin spelen IP, TCP en UDP een belangrijke rol.
  • Ook de applicaties spelen een rol. Deze kan je benaderen via een poort. Elke poort heeft een nummer en er zijn poortnummers die je moet kennen.
  • TCP/IP draait ook weer bovenop een ander protocol. Voorbeelden van deze protocollen kunnen zijn ethernet of PPP.
  • Ethernet gebruikt MAC-adresseringen om elkaar te kunnen vinden, daarboven gebruikt IP, IP-addressen als adressering.

netwerken

  • IP addressen gaan gepaard met een zogenaamd masker.
  • Op basis van het masker kan je bepalen in welk netwerk een PC of server zit.
  • Is er een PC die in hetzelfde IP netwerk zit als een andere PC, dan kunnen ze elkaar direct benaderen met een cross-kabel, hub of een switch.
  • Zitten de PC's in verschillende IP netwerken dan moet er een router (of gateway) geconfigureerd worden die ervoor kan zorgen dat de IP pakketten van de een naar de andere kant bezorgd worden.
LPIC-1 | Section 109.1 | Page 7/43

netwerken

  • Elk IP netwerk heeft een IP netwerk adres, een broadcast adres en meestal een gateway adres gedefineerd.
  • Het IP addres van de gateway mag je zelf bepalen zolang deze maar in hetzelfde netwerk zit.
  • Je kan met het programma 'ipcalc' op basis van een IP adres en een masker (of zogenaamd 'prefix') zelf uitrekenen wat het netwerk en het broadcast adres is.

netwerken

  • IP adressen kunnen lokaal zijn of globaal.
  • Locale IP adressen zijn niet schaars, maar op het internet worden ze niet geaccpeteerd. Je kan wel met je locale IP adres internetten door NAT (oftewel Network Adress Translation), waarbij het lokale adres omgekat wordt naar een globaal IP adres.
  • Locale IP adressen:
  • 10./8
  • 172.16/12
  • 192.168/16
  • Globale IP adressen lopen (grofweg) van 1 tot 239

netwerken

  • In het IP packet zet weer een ander pakketje.
  • Dit pakketje kan zijn 'TCP' of 'UDP'
  • TCP is een meer robuust protocol en zorgt dat pakketten herzonden worden als deze niet aankomen. TCP kan ook aan flow control doen.
  • UDP is een simpeler pakket, waardoor er meer en sneller gezonden kan worden ten koste van de betrouwbaarheid.
  • Daarnaast is er ook nog het ICMP protocol dat gebruikt wordt voor ping-verkeer.

  • TCP en UDP applicaties worden benaderd via poorten.

  • 110 pop3, 119 nntp, 139 netbios, 143 imap, 161 snmp
  • 443 https, 465 smtps, 993 imaps, 995 pop3s

ftp

  • FTP is het File Tranport Protocol.
  • Er zijn FTP programma's zoals het grafische FileZilla om meerdere files te gelijkertijd te kunnen downloaden/uploaden.
  • ftp staat standaard waarschijnlijk niet meer op de CLI geinstalleerd
  • Type je ftp dan krijg je een interactief menu, waar je commando's kunt geven.
  • Gebruik 'help' of '?' voor hulp en 'exit' of 'quit' om te stoppen.

ping

  • Met 'ping' kan je controlleren of een server wel up en running is.
  • Daarnaast geeft 'ping' ook informatie over hoeveel vertraging er is van naar die server(latency) is.
  • Let op dat sinds 'Windows Server 2008, de standaard instelling is dat er niet geluisterd wordt naar een ping. Je weet zo dus ook niet 100% zeker dat de server ook uit staat als je geen reacties krijgt.
  • Dit bovenstaande kan dus voor allerlei besturingssystemen en versies gelden.

telnet

  • Via telnet kan je naar een andere computer met een commando regel verbinden.
  • Het vervelende of gevaardlijke aan telnet is dat alle tekst blanko uit te lezen is (met een programma zoals Wireshark).
  • Beter is het om 'ssh' te gebruiken, wat hetzelfde bereikt maar wel veilig is.
  • Telnet staat nu ook standaart uit en we adviseren in prinicpe ook om het niet aan te zetten (in ieder geval liever niet in een productie omgeving).

tracepath

  • Met 'tracepath' kan je de verbinding naar een remote server zoeken.
  • Elke router die het verzonden pakket passeerd wordt weergegeven (mits deze routers dat niet blokkeren).
  • 'tracepath' is ideaal voor troubleshooten om te kijken waar een verbinding problemen heeft of tot waar een verbinding wel nog goed is.
  • Zie je 'no reply' dan heeft doet de router niet mee en wilt ook zijn traceroute niet weergeven.

What is the purpose of the dig command?
