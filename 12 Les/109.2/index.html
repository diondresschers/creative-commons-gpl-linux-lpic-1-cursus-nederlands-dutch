/etc/hostname

  • Je computer heeft niet alleen een IP adres maar ook een naam.
  • Deze wordt ook weergegenen in de prompt, bijvoorbeeld in '[dion@localhost ~]$ ', is de hostname 'localhost'
  • Je kan de hostname ook vinden via de environment variabele '$HOSTNAME'
  • Je kan de hostname zelf makkelijk wijzigen in de '/etc/hostname'.
  • De naam wordt weergegeven in deze file als 'hostname.domein'
  • Dit geldt officieel alleen voor de machine waarop je nu werkt (lokaal), hoe je aan de buitenkant weer bekend bent kan weer een ander verhaal zijn.

/etc/hosts

  • Om gebruik te maken van hostnamen wordt meestal gebruik gemaakt van een DNS (Domain Name Server). Deze zoekt het IP adres bij een hostnaam.
  • Echter kan je ook gebruik maken van een file die op je eigen PC staat. Dit is de '/etc/hosts' file.
  • In Windows heb je ook zo'n soort file en deze is vaak 'C:\Windows\System32\Drivers\Etc\hosts'
  • Je kan hier servers toevoegen die je voor persoonlijk gebruik vaker nodig kan hebben.
  • Het hangt van de instellingen af op de PC of er eerst de hosts file raadpleegt wordt of eerst de DNS server.

/etc/resolv.conf

  • Het belangrijkste van de /etc/resolv.conf file is om te bepalen welke DNS server gebruikt moet worden.
  • Uiteraard staat hier het IP adres van deze DNS server (en niet de naam).
  • De DNS server staat achter de term 'nameserver'
  • Je kan in principe per domein aangeven welke DNS server gebruikt moet worden.

/etc/nsswitch.conf

  • In '/etc/nsswitch' wordt bepaald naar welke Name Server eerste gekeken (geswitched) moet worden als er een servernaam moet worden opgezocht in.
  • Het is een lange file, maar het belangrijkste zit in de volgorde die wordt bepaald achter de tekst 'hosts:'
  • Bij bv: 'files dns myhostname' wordt eerst '/etc/hosts' gebruikt, dan de geconfigureerde DNS server en dan pas de lokaal ingestelde hostnaam.

ifconfig

  • Met 'ifconfig' laat je alle ingestellde interfaces zien.
  • Met 'ifconfig -a' laat je ook de interfaces zien die nog niet zijn ingesteld en anders verborgen blijven.
  • 'ifconfig' is eigenlijk al ouderwets en vervangen door het modernere 'ip address' of 'ip address show' van het nieuwere 'iproute2'
  • Je krijgt het MAC-addres, IPv4 alsmede IPv6 instellingen te zien.
  • Ook wordt informatie gegeven over het aantal pakketten dat verstuurd is met de eigenschappen daarvan.
  • Via 'ifconfig' kan je ook aanpassingen doen zoals b.v. het IP adres wijzigen, maar deze overleeft een reboot niet.

route

  • Met route zie je alle IP routes (netwerken) die een machine kent.
  • Moet er een IP pakket verstuurd worden, dan wordt aan de hand van deze 'route' bepaald naar welke interface of naar welke router deze gestuurd moet worden.
  • Je ziet in deze 'route' tabel naast netwerken ook default routes en Gateways, dit is al het verkeer dat niet in andere route/netwerk is bepaald.
  • De optie '-n' laat alleen Nummerieke IP addressen zien dat de leesbaarheid kan vergroten.
  • Je kan ook routes toevoegen (of weghalen) aan de 'route' met het 'route' commando, maar deze overleeft een reboot niet.

ping

  • Met ping kan je via het protocol 'icmp' controlleren of een server draait.
  • Ping kan ideaal zijn i.v.m. troubleshooten.
  • Als een server wel nog reageert op een 'ping' maar niet meer op 'ssh' kan de server toch vastgelopen zijn. Het reagereert wel nog op ping maar functioneert voor de rest niet (meer) goed.
  • Ping kan je doen naar een hostname of naar een IP adres.
