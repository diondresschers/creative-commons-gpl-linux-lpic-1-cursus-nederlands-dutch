ifconfig

  • Met ifconfig (nu 'ip route show') zie je instellingen van interfaces.
  • Gebruik de optie '-a' om alle interfaces te zien (ook degeen die nog niet geconfigureerd zijn).
  • Vergeet niet dat je met 'ifconfig' ook interfaces kunt aanpassen (niet permanent).

netstat

  • 'netstat' laat alle communicatie zien die de pc heeft met andere hosts.
  • Netstat is een zeer krachtige monitoring tool.
  • De optie '-t' geeft bv. het TCP verkeer weer, '-u' UDP verkeer.
  • Optie '-a' geeft al het verkeer weer en met '-p' wordt ook het programma weergegeven dat bij de poort hoort.
  • De routetabel kan je ook zien met 'netstat -r'.
  • 'netstat -i' laat de statistieken zien per interface.

/etc/init.d/network(ing)

  • Je kan de netwerkconfiguratie op verschillende manieren herstarten of bekijken
  • /etc/init.d/network status # Op RedHat
  • /etc/init.d/networking restart # Op Ubuntu
  • service network # met TAB-toets of 'reload' op RedHat/Ubuntu om de opties te zien'
  • systemctl status network # Voor systemen met 'systemd'

Vraag 1

Identify the statement that would create a default route using a gateway of 192.168.1.1.


  • netstat -add default gw
  • route default 192.168.1.1
  • ip route default 192.168.1.1
  • route add default gw 192.168.1.1
  • ifconfig default gw 192.168.1.1 eth0

Vraag 2

Suppose that the command netstat -a hangs for a long time without producing output. You might suspect:


  • a problem with NFS
  • a problem with DNS
  • a problem with NIS
  • a problem with routing
  • that the netstat daemon has crashed
