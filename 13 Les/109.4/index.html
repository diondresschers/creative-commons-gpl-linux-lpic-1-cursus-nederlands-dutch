/etc/hosts

  • Je kan voor je eigen computer een file aanpassen waardoor je in plaats van een IP adres, slechts een naam hoeft in te vullen.
  • Deze file is '/etc/hosts' en is meestal al gevuld met de naam 'localhost' voor het IP adres 127.0.0.1'
  • Deze file is vergelijkbaar met de local host file van Windows (meestal vindbaar in: 'C:\Windows\System32\Drivers\Etc\hosts').
  • Zie 109.2 voor meer informatie.

/etc/resolv.conf

  • In de '/etc/resolv.conf' kan je instellen bij welke DNS server je computer te raden gaan bij een DNS resolving
  • Je kan zelfs per domein bepalen welke DNS server gebruikt wordt.
  • Mocht het domein gelijk zijn dan wordt de DNS server gebruikt die boven de anderen staat.
  • Zie 109.2 voor meer informatie.

/etc/nsswitch.conf

  • In de file '/etc/nsswitch.conf' wordt bepaald welke systemen gebruikt worden om hostnamen op te zoeken en hier het IP adres van terug te geven.
  • De configuratiefile is lang, dus grep op '^hosts:' om te zien waar deze volgorde gedefinieerd zijn.
  • 'hosts: files dns' betekent: zoek eerst in de '/etc/hosts' file, staat hier niks vraag dan de DNS server.
  • Zie 109.2 voor meer informatie.

Vraag 1

You have a file /etc/resolv.conf, but the computer does not use the configured DNS servers to look up host names. What is most likely the problem?


  • The hosts entry in your /etc/nsswitch.conf does not list dns.
  • You do not have a /etc/named.conf file.
  • The localhost hostname is not properly configured in /etc/hosts.
  • The named daemon is not running on your computer.

Vraag 2

Which statement is true regarding the following /etc/resolv.conf file:

search example.com
127.0.0.1
208.77.188.166


  • There is a syntax error
  • If DNS queries to the localhost fail, the server 208.77.188.166 will be queried
  • example.com will be appended to all host lookups
  • The DNS servers at 127.0.0.1 and 208.77.188.166 will be queried in a round robin fashion
